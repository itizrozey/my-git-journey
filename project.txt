
Inventory Management System for an automobile system/warehouse:

    System Overview:
        The system will manage the inventory of automobile parts/products in a ware house, tracking products with attributes

        name: Product name (i.e, engin oil, break pads, etc)
        category: Product category (e.g, Engine, Breakes, etc)
        quantity: Current stock quantity
        price: Product price
        supplier: Product supplier

        The system will have the following features: 
        1. Add stock: Increase the quantity of a product in stock
        2. Sell Items: Reduce the quantity of a product in stock
        3. Low-Stock Alerts: Notify when a product's quantity falls below a certain threshold
        4. Total Inventory Value: Calculate the total value of all products in stock
        5. Sales Report by Period: Generate a report of sales for a given period (e.g., daily, weekly, monthly) 




 

    *System Design* Here's a possible design for the system:
     1. *Product Class*: 
     - `__init__`: Initialize a product with name, category, quantity, price, and supplier 
     - `__str__`: Return a string representation of the product 
    
     2. *Inventory Class*: 
     - `__init__`: Initialize an empty inventory 
     - `add_product`: Add a product to the inventory 
     - `add_stock`: Increase the quantity of a product in stock 
     - `sell_item`: Reduce the quantity of a product in stock 
     - `check_low_stock`: Check if a product's quantity is below the threshold 
     - `total_inventory_value`: Calculate the total value of all products in stock 
     - `sales_report`: Generate a sales report for a given period 
     
     3. *Database*:
      - Store product data in a database (e.g., SQLite, "MySQL") 
      - Use a database to persist data between sessions
      
       *Database Schema* Here's a possible database schema:
        CREATE TABLE products ( 
            id INTEGER PRIMARY KEY, 
            name TEXT NOT NULL, 
            category TEXT NOT NULL, 
            quantity INTEGER NOT NULL, 
            price REAL NOT NULL, 
            supplier TEXT NOT NULL ); 
            
        CREATE TABLE sales ( 
            id INTEGER PRIMARY KEY, 
            product_id INTEGER NOT NULL,
             date DATE NOT NULL, 
             quantity INTEGER NOT NULL, 
             total REAL NOT NULL, 
             FOREIGN KEY (product_id) REFERENCES products (id) );
